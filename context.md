Objective
Process-driven inventory tracking system
Products move through multiple manufacturing stages
Each stage is handled by a different team
Teams enter: Input/output quantity
A specific role handles rejections
Items are grouped by Brand → Items

Manufacturing Process Flow
The product moves sequentially through these stages:

Melting
Shot Blast
Inspection
Proof Machines
CNC
Hardness Inspection
Balancing
Packing
Melting

Data Entry Point: 1
The user will:
View the listed Brand–Item entries
Enter the Production Quantity (units successfully processed)
Review and Submit the entered details

Shot Blast
Data Entry Point: 2
The user will:
View the listed Brand–Item entries
Enter the Input Quantity (units taken for processing)
The input quantity will be compared with the Melting production quantity, and the system will
alert if there is any mismatch
Enter the Production Quantity (units successfully processed)
Review and Submit the entered details

Inspection
Data Entry Point: 3
The user will:
View the listed Brand–Item entries
Enter the Input Quantity (units taken for inspection)
The input quantity will be compared with the shot blast production quantity, and the system will
alert if there is any mismatch
Enter the Rejection Quantity (units that failed inspection)
Review and Submit the entered details
Note: Approved Quantity will be auto-calculated by the system

Post-Inspection – Outward Transfer to External/Internal Locations
Data Entry Point: 4
After the Inspection stage, the approved items may be sent to different external processing
locations
The user will:
Select the Destination Location
Enter the Quantity dispatched
Review and Submit the details
